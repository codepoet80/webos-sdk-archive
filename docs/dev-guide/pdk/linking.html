Linking for the Device

Using the -l command option, apps should not link to, or call functions from, libraries that are not on the following whitelist of libraries that came with the PDK:

SDL 

libSDL.so, libSDL-1.2.so.0
libSDL_image.so, libSDL_image-1.2.so.0
libSDL_mixer.so, libSDL_mixer-1.2.so.0
libSDL_net.so, libSDL_net-1.2.so.0
libSDL_ttf.so, libSDL_ttf-2.0.so.0

PDL 

libpdl.so
libSDL_cinema.so

OpenGL ES 

libGLES_CM.so
libGLESv2.so

glibc (libdl explicitly not included) 

libc.so, libc.so.6
libgcc_s.so, libgcc_s.so.1
libm.so, libm.so.6
libpthread.so, libpthread.so.0
librt.so, librt.so.1
libcrypt.so, libcrypt.so.1
libutil.so, libutil.so.1

libstdc++ 

libstdc++.so, libstdc++.so.6

libz (added January 2011) 

libz.so, libz.so.1

openal (added January 2011) 

libopenal.so, libopenal.so.1

libcurl (added July 2011) 

libcurl.so.4

openssl 0.9.8 (added July 2011) 

libcrypto.so, libcrypto.so.0.9.8
libssl.so, libssl.so.0.9.8

libpng 1.2 (added July 2011) 

libpng12.so.0

libfreetype 2 (added July 2011) 

libfreetype.so, libfreetype.so.6

This guarantees your apps will run on the device and be forward-compatible with future versions of device run-time libraries. Note that this is enforced through the app submission checker.
