Resource Monitor

The webOS Resource Monitor is a tool that graphs your application's DOM node count and open service handles, helping you determine your app's memory footprint and if it is leaking memory.

If your app leaves DOM nodes open after the app or scene is closed, then it is leaking memory. Typically, event handlers that remain open after a scene or app is closed are the cause.


Using the webOS Resource Monitor

To start the webOS Resource Monitor, run it from the command line, specifying the app id: 

palm-worm com.mystuff.hello

You can then open your app in the Emulator, and watch as its data is graphed. You should test intensive app operation, checking for memory leaks.

  • When the app is closed, memory should return to zero.

  • When dismissing a notification in a headless app, memory should return to the baseline level the app uses in the background. This baseline should remain constant.
